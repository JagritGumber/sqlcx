## [0.2.0] — Full driver parity + INNER JOIN support

First release of the Rust-based sqlcx. Previous npm `sqlcx-orm@0.1.2` was
the TypeScript codebase; this version is the full rewrite with every
language in feature parity.

### Added

- **Driver parity across all languages.** Every supported language now has
  Postgres, MySQL, and SQLite drivers. 12 drivers total:
  - TypeScript: `pg`, `bun-sql`, `mysql2`, `better-sqlite3`
  - Python: `psycopg`, `asyncpg`, `sqlite3`, `mysql-connector`
  - Rust: `sqlx` (Postgres), `sqlx-mysql`, `sqlx-sqlite`, `tokio-postgres`
  - Go: `database-sql` (Postgres), `database-sql-mysql`, `database-sql-sqlite`, `pgx`
- **INNER JOIN support** across all three SQL dialects (postgres, mysql,
  sqlite parsers). Qualified columns (`SELECT users.id, orgs.slug FROM users INNER JOIN orgs ON ...`)
  are typed correctly. Colliding column names without explicit `AS` aliases
  are rejected at parse time with a clear error and suggested rewrite.
- Per-language `TsTypeMap` and `PyTypeMap` traits enabling driver-specific
  type overrides (e.g. SQLite `bool → int`, `date → str`; MySQL `Binary → Buffer`).
- `SqlxBackend` and `DatabaseSqlBackend` enums parameterizing the Rust and
  Go generators over Postgres / MySQL / SQLite so a single generator handles
  all three backends.
- CI workflow (`.github/workflows/ci.yml`): `cargo fmt`, `cargo build`,
  `cargo test --workspace`, `cargo clippy -- -D warnings`, TypeScript
  typecheck integration tests with Node + tsc.
- Dependabot config for cargo, npm, and GitHub Actions.
- Issue templates (bug, feature, driver-request) and PR template.
- `parser::joins` module with `AliasMap`, `parse_join_clauses`,
  `resolve_multi_table_columns`, `has_outer_join`, and
  `resolve_multi_table_select_column`.
- SQL literal-aware placeholder rewriting — `$N` inside single-quoted
  string literals (e.g. `SELECT '$1' FROM x`) is preserved, not rewritten.
  `''` SQL escape handled correctly.
- Reused `$N` params (e.g. `WHERE x = $1 OR y = $1`) generate the correct
  number of bind/arg calls in positional order for MySQL/SQLite backends.
  Non-Copy Rust types (e.g. `serde_json::Value`) clone on every bind to
  avoid use-after-move compile errors.

### Changed

- Rust edition bumped from `2021` to `2024`. All `let gen = ...` test
  bindings renamed to `let gen_ = ...` (`gen` is a reserved keyword in
  the 2024 edition). Clippy `collapsible_if` lint fixes applied where it
  now fires.
- Per-language `<lang>/common/` modules consolidate the shared surface
  across drivers (type mapping, row/params class generation, SQL escape,
  file grouping). New drivers in the same language implement a thin type
  map plus driver-specific client content — typically ~80-150 LOC instead
  of the prior ~250-300.
- TypeScript pg + bun-sql drivers fully share `generate_query_function`
  via `common/query_fn.rs` — both drivers fit the async DatabaseClient
  shape exactly.
- Version label corrected from a misleading `1.0.0` (never published) to
  `0.1.3`. Previous npm `sqlcx-orm@0.1.2` was the old TypeScript codebase.
- LICENSE copyright year updated to `2025-2026`.
- mysql-connector driver escapes literal `%` to `%%` to avoid printf-style
  format specifier crashes in `cursor.execute(sql, params)`.

### Fixed

- `split_as_alias` case-insensitive `AS` parsing no longer panics on
  non-ASCII identifiers. Uses a regex against the original string for
  byte-boundary safety.
- `UNSUPPORTED_JOIN_RE` now catches `NATURAL INNER JOIN` and other
  `<modifier> INNER|OUTER JOIN` forms.
- `has_outer_join` scopes the JOIN keyword check to the outer FROM body
  via `FROM_CLAUSE_RE`. Queries with JOINs inside subqueries (e.g.
  `WHERE id IN (SELECT ... JOIN ...)`) correctly take the single-table
  path on the outer query.
- Generated code banner (`// Code generated by sqlcx. DO NOT EDIT.`)
  normalized across all TypeScript drivers.
- Pipeline cache correctly invalidates on driver/schema/overrides changes.
