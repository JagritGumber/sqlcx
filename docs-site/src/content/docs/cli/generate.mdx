## Usage

```bash
sqlcx generate
```

Runs the full code-generation pipeline. Reads your `sqlcx.toml`, scans SQL files, parses schema and queries, then writes typed output files for every configured target.

---

## Pipeline

<Steps>

1. **Load config** — reads `sqlcx.toml` (or `sqlcx.json`) from the current directory.

2. **Glob SQL files** — recursively finds every `*.sql` file under the configured `sql` directory.

3. **Partition** — splits files into schema files and query files:
   - **Schema files**: any `.sql` file that is NOT inside a `queries/` subdirectory and does NOT contain a `-- name:` annotation.
   - **Query files**: any `.sql` file inside a `queries/` subdirectory OR containing at least one `-- name:` annotation.

4. **Hash** — computes a SHA-256 hash over all file contents.

5. **Cache check** — looks for a matching entry in `.sqlcx/`. If a cache hit is found, skips parsing entirely.

6. **Parse** — on a cache miss, invokes the configured SQL parser to build the Intermediate Representation (IR): tables, enums, and queries.

7. **Write cache** — stores the new IR to `.sqlcx/` keyed by the content hash.

8. **Generate** — for each `[[targets]]` block in config, calls the matching language plugin to produce output files.

9. **Write output** — writes generated files to the `out` directory for each target.

</Steps>

## Example output

```
$ sqlcx generate
Loading config from /myapp
Scanning SQL files in /myapp/sql
Cache miss — parsing SQL files
Writing /myapp/src/generated/users.ts
Writing /myapp/src/generated/schema.ts
Done.
```

On a subsequent run with no SQL changes:

```
$ sqlcx generate
Loading config from /myapp
Scanning SQL files in /myapp/sql
Cache hit — using cached IR
Writing /myapp/src/generated/users.ts
Writing /myapp/src/generated/schema.ts
Done.
```

All pipeline messages are written to **stderr**. Only generated file content goes to the output files on disk.

---

## File classification rules

| File | Classification |
|---|---|
| `sql/schema.sql` | Schema |
| `sql/users.sql` | Schema (no `-- name:`, not in `queries/`) |
| `sql/queries/users.sql` | Query (inside `queries/`) |
| `sql/find_user.sql` containing `-- name: FindUser :one` | Query (has annotation) |

A file with a `-- name:` annotation is always treated as a query file regardless of its directory.
