## What's Planned

sqlcx will generate Python code from your SQL queries, with first-class support for **Pydantic models** and the **asyncpg** driver.

The design follows the same SQL-first philosophy: write SQL, get typed Python functions back. No ORM, no schema DSL — just your queries, typed.

### Planned Stack

- **Schema generation** — Pydantic v2 models for each table and query result
- **Driver** — asyncpg for PostgreSQL, aiosqlite for SQLite
- **Async-first** — all generated functions use `async/await`
- **Validation** — optional Pydantic field validators on results

## What Generated Python Will Look Like

Given this SQL query:

```sql
-- name: get_user_by_email :one
SELECT id, email, created_at
FROM users
WHERE email = $1;
```

sqlcx will generate:

```python
# generated by sqlcx — do not edit

from __future__ import annotations
from datetime import datetime
from pydantic import BaseModel
import asyncpg


class GetUserByEmailRow(BaseModel):
    id: int
    email: str
    created_at: datetime


async def get_user_by_email(
    conn: asyncpg.Connection,
    email: str,
) -> GetUserByEmailRow | None:
    row = await conn.fetchrow(
        """
        SELECT id, email, created_at
        FROM users
        WHERE email = $1
        """,
        email,
    )
    if row is None:
        return None
    return GetUserByEmailRow.model_validate(dict(row))
```

For list queries (`:many`), the return type becomes `list[GetUserByEmailRow]`.
