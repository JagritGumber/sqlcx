sqlcx reads your schema and generates two structs per table: a **select struct** for query results and an **insert struct** for new rows.

---

## Configuration

```toml
[[targets]]
language = "go"
schema   = "sql/schema.sql"
out      = "internal/db"
```

The `schema` key selects struct-only output (no driver). Pair it with `driver = "database-sql"` if you also want typed query functions.

---

## SQL → Go struct

Given this schema:

```sql
CREATE TYPE user_status AS ENUM ('active', 'inactive', 'banned');

CREATE TABLE users (
  id         SERIAL PRIMARY KEY,
  name       TEXT        NOT NULL,
  email      TEXT        NOT NULL UNIQUE,
  bio        TEXT,
  status     user_status NOT NULL DEFAULT 'active',
  tags       TEXT[],
  created_at TIMESTAMP   NOT NULL DEFAULT NOW()
);
```

sqlcx generates:

```go
// Code generated by sqlcx. DO NOT EDIT.
package db

import (
	"time"
)

type Users struct {
	Id        int64      `db:"id"         json:"id"`
	Name      string     `db:"name"       json:"name"`
	Email     string     `db:"email"      json:"email"`
	Bio       *string    `db:"bio"        json:"bio"`
	Status    string     `db:"status"     json:"status"`
	Tags      []string   `db:"tags"       json:"tags"`
	CreatedAt time.Time  `db:"created_at" json:"created_at"`
}

// InsertUsers has optional fields for columns with defaults
type InsertUsers struct {
	Name      string    `db:"name"   json:"name"`
	Email     string    `db:"email"  json:"email"`
	Bio       *string   `db:"bio"    json:"bio"`
	Status    *string   `db:"status" json:"status"`
	Tags      *[]string `db:"tags"   json:"tags"`
}
```

## Type mapping

| SQL type | Go type (non-nullable) | Go type (nullable) |
|----------|------------------------|---------------------|
| `TEXT`, `VARCHAR` | `string` | `*string` |
| `INTEGER`, `SERIAL` | `int64` | `*int64` |
| `BIGINT`, `BIGSERIAL` | `int64` | `*int64` |
| `REAL`, `FLOAT`, `NUMERIC` | `float64` | `*float64` |
| `BOOLEAN` | `bool` | `*bool` |
| `TIMESTAMP`, `DATE` | `time.Time` | `*time.Time` |
| `JSONB`, `JSON` | `json.RawMessage` | `*json.RawMessage` |
| `BYTEA` | `[]byte` | `*[]byte` |
| `TEXT[]` (array) | `[]string` | `*[]string` |
| Enum type | `string` | `*string` |

---

## Field naming

SQL column names are converted to **PascalCase** for Go field names:

| SQL column | Go field |
|-----------|----------|
| `id` | `Id` |
| `user_id` | `UserId` |
| `created_at` | `CreatedAt` |
| `email_verified` | `EmailVerified` |

Both `db` and `json` struct tags are always emitted, using the original SQL column name.

---

## Nullable columns

Columns declared without `NOT NULL` become pointer types (`*T`). This mirrors Go's idiomatic approach for optional values and avoids `sql.NullString` / `sql.NullInt64`.

```sql
bio TEXT   -- no NOT NULL → *string in Go
```

```go
Bio *string `db:"bio" json:"bio"`
```

---

## Insert structs

The `InsertUsers` struct omits auto-managed columns (`id`, `created_at`) and makes columns with defaults optional (`*T`):

- `id` — excluded (auto-increment)
- `created_at` — excluded (server default)
- `status` — `*string` (has default `'active'`)
- `name`, `email` — `string` (required, no default)

---

## Imports

sqlcx adds only the imports your types require:

- `time` — when any column maps to `time.Time`
- `encoding/json` — when any column maps to `json.RawMessage`

No other dependencies are introduced.
