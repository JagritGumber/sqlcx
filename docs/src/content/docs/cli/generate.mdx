---
title: generate
description: Generate type-safe code from your SQL files.
---

```bash
npx sqlcx generate
```

Reads your SQL, generates typed code. That's the whole command.

---

## What happens

1. sqlcx reads `sqlcx.toml` from your project root
2. Finds all `.sql` files in your `sql` directory
3. Parses your tables, enums, and annotated queries
4. Writes typed code to each target's `out` directory

```
$ npx sqlcx generate
Writing src/generated/schema.ts
Writing src/generated/client.ts
Writing src/generated/users.queries.ts
Done.
```

If your SQL hasn't changed since the last run, parsing is skipped automatically (cached). The output files are still regenerated.

---

## What goes where

sqlcx figures out which files are schema definitions and which are queries:

- **Schema** — `CREATE TABLE`, `CREATE TYPE` statements. Any `.sql` file without `-- name:` annotations.
- **Queries** — Files with `-- name:` annotations, or anything inside a `queries/` subdirectory.

```
sql/
├── schema.sql              ← schema (CREATE TABLE, CREATE TYPE)
└── queries/
    ├── users.sql           ← queries (-- name: GetUser :one)
    └── posts.sql           ← queries (-- name: ListPosts :many)
```

## Adding to your workflow

**package.json:**
```json
{
  "scripts": {
    "generate": "sqlcx generate",
    "build": "sqlcx generate && tsc"
  }
}
```

**Makefile:**
```makefile
generate:
	sqlcx generate

build: generate
	go build ./...
```

Run it before your build step. If SQL hasn't changed, it's near-instant.
