SQLite is loosely typed ‚Äî everything is stored as one of five affinities (NULL, INTEGER, REAL, TEXT, BLOB). sqlcx maps declared types to their affinities and follows community conventions for booleans and datetimes.

```toml
parser = "sqlite"
```

---

## What's supported

| Feature | Example |
|---------|---------|
| Type affinity mapping | `INTEGER`, `REAL`, `TEXT`, `BLOB` |
| Auto-increment | `INTEGER PRIMARY KEY AUTOINCREMENT` |
| `?` parameters | Positional, in order |
| Default values | `DEFAULT 0`, `DEFAULT CURRENT_TIMESTAMP` |
| NOT NULL | Full constraint parsing |
| Boolean convention | `BOOLEAN` ‚Üí boolean (stored as 0/1) |
| Datetime convention | `DATETIME` ‚Üí date (stored as TEXT/INTEGER) |

---

## Type mapping

| SQL Type / Affinity | IR Category | Notes |
|---------------------|-------------|-------|
| `INTEGER`, `INT`, `BIGINT`, `SMALLINT` | number | INTEGER affinity |
| `REAL`, `DOUBLE`, `FLOAT` | number | REAL affinity |
| `TEXT`, `VARCHAR`, `CHAR`, `CLOB` | string | TEXT affinity |
| `BOOLEAN` | boolean | Convention ‚Äî stored as 0/1 |
| `DATETIME`, `DATE`, `TIMESTAMP` | date | Convention ‚Äî stored as TEXT or INTEGER |
| `JSON` | json | Convention ‚Äî stored as TEXT |
| `BLOB` | binary | BLOB affinity |
| No type / `NUMERIC` | unknown | NUMERIC affinity, no clear mapping |

---

## Example

```sql
CREATE TABLE tasks (
  id        INTEGER PRIMARY KEY AUTOINCREMENT,
  title     TEXT NOT NULL,
  completed BOOLEAN NOT NULL DEFAULT 0,
  due_date  DATETIME,
  priority  INTEGER NOT NULL DEFAULT 1,
  notes     TEXT
);
```

sqlcx maps `completed` to boolean and `due_date` to datetime by convention, even though SQLite stores both as integers or text internally.

---

## Notes

**No ENUMs** ‚Äî SQLite has no native enum type. Use `TEXT` with a `CHECK` constraint if you need enumerated values. sqlcx will map the column to string.

**BOOLEAN is 0/1** ‚Äî SQLite stores booleans as integers. sqlcx recognizes the `BOOLEAN` type name and maps it accordingly, but at the database level it's always an integer.

**DATETIME is TEXT** ‚Äî SQLite stores dates as ISO 8601 text, Julian day numbers, or Unix timestamps. sqlcx maps `DATETIME`/`DATE`/`TIMESTAMP` to the date category by convention.

**AUTOINCREMENT** ‚Äî One word, no underscore. `INTEGER PRIMARY KEY` alone also auto-increments in SQLite (without the keyword). sqlcx handles both forms.
