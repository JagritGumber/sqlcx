## The problem with modern ORMs

Every popular ORM ships with trade-offs that compound at scale.

**Prisma** bundles a 1.6 MB query engine binary and a JavaScript runtime on top of it. Your Lambda cold start pays for it every time.

**Drizzle** is lighter, but its deep TypeScript generics slow down IDE type-checking on large schemas — the more tables you add, the more your editor lags.

**sqlc** generates type-safe code from plain SQL, but it only targets Go. TypeScript, Rust, and Python developers are left out.

---

## The sqlcx approach

sqlcx treats SQL as the single source of truth and generates idiomatic, typed client code for every target language — with no runtime library required.

- **SQL is the schema.** You write `CREATE TABLE` and named queries. sqlcx reads them.
- **Zero runtime.** Generated code calls your database driver directly. There is nothing between your query and the wire.
- **Every language.** One tool, one config, multiple output targets.
- **Open validation.** Pick TypeBox, Zod, or Serde — sqlcx plugs into the ecosystem you already use.

> If you know SQL, you already know sqlcx.

| **sqlcx** | **0 KB** |

sqlcx generates code. There is nothing to ship at runtime.

## Same query, every language

Write your SQL query once:

```sql
-- name: GetUser :one
SELECT id, name, email, status
FROM users
WHERE id = $1;
```

Use the generated code in any language:

<Tabs>
  <TabItem label="TypeScript">
    ```typescript
    import { getUser } from './generated/users';
    import { sql } from 'bun';

    const user = await getUser(sql, { id: 42 });
    console.log(user.name);
    ```
  </TabItem>
  <TabItem label="Go">
    ```go
    import "github.com/example/myapp/generated"

    user, err := queries.GetUser(ctx, 42)
    if err != nil {
        return err
    }
    fmt.Println(user.Name)
    ```
  </TabItem>
  <TabItem label="Rust">
    ```rust
    use crate::generated::get_user;

    let user = get_user(&pool, 42).await?;
    println!("{}", user.name);
    ```
  </TabItem>
</Tabs>

Every version is fully typed from your schema. No hand-written interfaces. No runtime surprises.
