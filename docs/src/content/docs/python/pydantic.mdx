title: Pydantic
description: Generate Pydantic v2 BaseModel classes from your SQL tables with full type annotations.
---

sqlcx generates a `models.py` file containing Pydantic v2 `BaseModel` classes for every table in your schema. Each model has typed fields derived directly from your SQL column definitions — no manual type mapping required.

---

## Configuration

Set `language = "python"` and `schema = "pydantic"` in your target block:

```toml
[[targets]]
language = "python"
out      = "src/generated"
schema   = "pydantic"
driver   = "none"
```

Install Pydantic:

```bash
pip install pydantic
# requires pydantic >= 2.0
```

## Generated output

Given this schema:

```sql
CREATE TYPE user_status AS ENUM ('active', 'inactive', 'banned');

CREATE TABLE users (
  id         SERIAL      PRIMARY KEY,
  name       TEXT        NOT NULL,
  email      TEXT        NOT NULL UNIQUE,
  bio        TEXT,
  status     user_status NOT NULL DEFAULT 'active',
  tags       TEXT[],
  created_at TIMESTAMP   NOT NULL DEFAULT NOW()
);
```

sqlcx writes `models.py`:

```python
# Code generated by sqlcx. DO NOT EDIT.

from datetime import datetime
from enum import Enum
from pydantic import BaseModel, ConfigDict

class UserStatus(str, Enum):
    ACTIVE = "active"
    INACTIVE = "inactive"
    BANNED = "banned"

class SelectUsers(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    id: int
    name: str
    email: str
    bio: str | None
    status: UserStatus
    tags: list[str] | None
    created_at: datetime

class InsertUsers(BaseModel):
    model_config = ConfigDict(from_attributes=True)

    name: str
    email: str
    id: int | None = None
    bio: str | None = None
    status: UserStatus | None = None
    tags: list[str] | None = None
    created_at: datetime | None = None
```

## How types are derived

### Select models

Each column maps to a Python type. Nullable columns use union syntax:

```python
# NOT NULL column
name: str

# Nullable column
bio: str | None
```

### Insert models

Columns with a `DEFAULT` value or that are nullable get `= None`:

```python
# Required, no default
name: str

# Has DEFAULT — optional on insert
status: UserStatus | None = None

# Nullable — optional on insert
bio: str | None = None
```

Required fields are always listed before optional fields to satisfy Python syntax rules.

## Type mapping

| SQL type | Python type |
|----------|------------|
| `text`, `varchar`, `char` | `str` |
| `integer`, `serial`, `bigint` | `int` |
| `real`, `float`, `double`, `decimal`, `numeric` | `float` |
| `boolean` | `bool` |
| `timestamp`, `timestamptz` | `datetime` |
| `date` | `date` |
| `time` | `time` |
| `json`, `jsonb` | `Any` |
| `uuid` | `str` |
| `bytea` | `bytes` |
| `text[]`, `integer[]` | `list[...]` |
| enum type | `str(Enum)` class |

---

## Enum handling

SQL enums are generated as Python `Enum` classes that inherit from both `str` and `Enum`:

```python
class UserStatus(str, Enum):
    ACTIVE = "active"
    INACTIVE = "inactive"
    BANNED = "banned"
```

Enum values are sanitized into valid Python identifiers:
- Hyphens become underscores: `"in-progress"` → `IN_PROGRESS`
- Python keywords get a trailing underscore: `"class"` → `CLASS_`
- Leading digits get a prefix: `"123abc"` → `_123ABC`

## ORM compatibility

Every generated model includes `model_config = ConfigDict(from_attributes=True)`, which means you can construct models directly from database row objects:

```python
import asyncpg

row = await conn.fetchrow("SELECT * FROM users WHERE id = $1", user_id)
user = SelectUsers.model_validate(dict(row))
```

This works with `asyncpg.Record`, `psycopg` rows, SQLAlchemy `Row` objects, and any mapping type.

## Overrides

Use `overrides` to change how a specific SQL type maps to a Python type:

```toml
[[targets]]
language = "python"
out      = "src/generated"
schema   = "pydantic"
driver   = "none"

[targets.overrides]
uuid = "str"
```

Supported override values: `"str"`, `"int"`, `"float"`, `"bool"`, or any Python type name.
