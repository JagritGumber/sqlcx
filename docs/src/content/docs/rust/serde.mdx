sqlcx reads your schema and generates Serde-compatible Rust structs, one per table. Each struct derives `Debug`, `Clone`, `Serialize`, `Deserialize`, and `sqlx::FromRow`.

---

## Configuration

```toml
[[targets]]
language = "rust"
schema   = "sql/schema.sql"
out      = "src/generated"
```

The `schema` key selects struct-only output. Add `driver = "sqlx"` to also generate typed query functions.

---

## SQL → Rust struct

Given this schema:

```sql
CREATE TYPE user_status AS ENUM ('active', 'inactive', 'banned');

CREATE TABLE users (
  id         SERIAL PRIMARY KEY,
  name       TEXT        NOT NULL,
  email      TEXT        NOT NULL UNIQUE,
  bio        TEXT,
  status     user_status NOT NULL DEFAULT 'active',
  tags       TEXT[],
  created_at TIMESTAMP   NOT NULL DEFAULT NOW()
);
```

sqlcx generates:

```rust
// Code generated by sqlcx. DO NOT EDIT.

use serde::{Deserialize, Serialize};
use chrono;

#[derive(Debug, Clone, Serialize, Deserialize, sqlx::FromRow)]
pub struct Users {
    pub id: i32,
    pub name: String,
    pub email: String,
    pub bio: Option<String>,
    pub status: String,
    pub tags: Option<Vec<String>>,
    pub created_at: chrono::NaiveDateTime,
}

/// Insert type for users — columns with defaults become Option<T>
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct InsertUsers {
    pub id: Option<i32>,
    pub name: String,
    pub email: String,
    pub bio: Option<String>,
    pub status: Option<String>,
    pub tags: Option<Vec<String>>,
    pub created_at: Option<chrono::NaiveDateTime>,
}
```

## Type mapping

| SQL type | Rust type (non-nullable) | Rust type (nullable) |
|----------|--------------------------|----------------------|
| `TEXT`, `VARCHAR`, `UUID` | `String` | `Option<String>` |
| `INTEGER`, `SERIAL` | `i32` | `Option<i32>` |
| `BIGINT`, `BIGSERIAL` | `i64` | `Option<i64>` |
| `REAL`, `FLOAT`, `NUMERIC` | `f64` | `Option<f64>` |
| `BOOLEAN` | `bool` | `Option<bool>` |
| `TIMESTAMP` | `chrono::NaiveDateTime` | `Option<chrono::NaiveDateTime>` |
| `DATE` | `chrono::NaiveDate` | `Option<chrono::NaiveDate>` |
| `TIME` | `chrono::NaiveTime` | `Option<chrono::NaiveTime>` |
| `JSONB`, `JSON` | `serde_json::Value` | `Option<serde_json::Value>` |
| `BYTEA` | `Vec<u8>` | `Option<Vec<u8>>` |
| `TEXT[]` (array) | `Vec<String>` | `Option<Vec<String>>` |
| Enum type | `String` | `Option<String>` |

## Field naming

Rust field names keep the original **snake_case** SQL column names exactly. No conversion is applied.

| SQL column | Rust field |
|-----------|------------|
| `id` | `id` |
| `user_id` | `user_id` |
| `created_at` | `created_at` |
| `email_verified` | `email_verified` |

---

## Nullable columns

Columns without `NOT NULL` become `Option<T>`:

```sql
bio TEXT   -- no NOT NULL → Option<String>
```

```rust
pub bio: Option<String>,
```

---

## Insert structs

`InsertUsers` is a separate struct for new rows. It includes all columns, but columns with a `DEFAULT` expression become `Option<T>` — you can pass `None` and let the database fill them in.

The insert struct does **not** derive `sqlx::FromRow` since it is not read back from the database.

---

## Imports

sqlcx adds only the imports your types require:

- `use serde::{Deserialize, Serialize};` — always emitted
- `use chrono;` — when any column maps to a chrono type
- `use serde_json;` — when any column maps to `serde_json::Value`

---

## Cargo dependencies

Add these to `Cargo.toml`:

```toml
[dependencies]
serde       = { version = "1", features = ["derive"] }
serde_json  = "1"            # only if you have JSON/JSONB columns
chrono      = { version = "0.4", features = ["serde"] }   # only if you have date columns
sqlx        = { version = "0.8", features = ["postgres", "chrono", "runtime-tokio"] }
```
